Development of Climb the Tower

Overview

Climb the Tower is a mobile game I developed and published on Google Play. It was developed in Gamemaker Studio 2 which uses an object oriented language. I created this app to gain experience and a deeper understanding of how to structure an app. Within a 10-day timeframe I created the code, graphics and UI for this application.

Dividing development into pieces

In order to create this app within the timeframe I split development into key parts:

  • Day 1: Plan out the general idea of the game. This includes: gameplay, aesthetic and goals.
  • Day 2: Code a prototype.
  • Day 3: Refine gameplay.
  • Day 4: Design and program UI elements. This includes: main menu, options and buttons.
  • Day 5: Design game graphics. This includes: backgrounds, characters and UI visuals.
  • Day 6: Program characters and their different abilites.
  • Day 7: Program different gamemodes and a shop where the player can unlock the characters.
  • Day 8: Implement mobile advertisements with the Google Admob API.
  • Day 9: Fix bugs and finalize project.
  • Day 10: Publish the application on Google Play.

Following this list I had a plan and could use my time efficiently.

Rapid prototyping

During application developement it is vital to create a functioning prototype as early as possible. Sometimes an idea sounds good on paper but is flawed in practice. Building a quick prototype that showcases the basic functionalities of the app can help the developer recognise problems in the early stages.

Climb the Tower is a game of infinite stairs. The user has to use 2 buttons to climb the Tower's platforms in order to get a high score. The main challenge was to create a way to easily generate platforms for the player to climb. I achieved this using a script that manages a list containing the 6 next platforms. The reason I chose the number 6 is because that is the amount of platforms rendered on the screen. Every time the user presses a button to jump the list gets updated. The last platform gets 'poped' from the list and a new one is added on top. In order to randomize the position every platform is assigned either the number 1 or -1. This number defines if it would be on the left or right of the previous platform. This approach has several benefits:

  • By tracking only 6 platforms we do not have a problem with memory management. This is important as the game can technically go on forever.
  • The platform list only has to track a single value: 1 or -1. This makes understanding the game state very simple.

In the above image we can see the platforms aswell as a bronze coin. If the player reaches the coin he gains a point. After collecting enough coins he can unlock new characters in the shop. To add coins I just had to update the information stored in platform list. Instead of just storing the 1 or -1 value I also stored an additional value representing the reward on the platform.

Character Design

These are some of the characters I designed. It is importand for the character to stand out in the game so the player can easily distinguish their position in the game. Every character has a unique ability and animation.

Google Admob

The game uses Google Admob to serve advertisments and monetise the application. Interstitial ads are served every 4th time the player fails. After every failed attempt the player is given the option to watch a rewarded ad that doubles the coins he collected. A Async event is called after the ad is complete and the coins are added to his account.
